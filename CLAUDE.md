# 주식 기법 플레이북 — 콘텐츠 규칙

한국·미국 주식 매매 기법 + 거시·경제지표 학습 사이트. Astro 5 정적 사이트, GitHub Pages 배포.

## 콘텐츠 추가/수정 방법

- 글 1편 = `src/content/techniques/<slug>.md` 파일 1개. slug는 영문 kebab-case.
- frontmatter 스키마는 `src/content.config.ts`가 기준. **frontmatter 값은 JSON 리터럴 형식으로 한 줄씩** 쓴다 (문자열은 큰따옴표, charts는 한 줄 JSON 배열).
- 카테고리(8개 고정): 차트 패턴 / 보조지표 / 추세·모멘텀 / 가치·성장 투자 / 퀀트·시스템 / 수급·이벤트 / 거시·경제지표 / 리스크·심리. 새 카테고리는 `content.config.ts`와 `src/utils/site.ts` 두 곳을 함께 수정해야 한다.
- 검토 전 초안은 `draft: true`로 두면 사이트에 노출되지 않는다.

## 글 구조

**매매 기법 글**: `## 개요와 원리 → ## 핵심 규칙(### 진입/### 청산과 손절) → ## 실전 적용 → ## 장단점(표) → ## 흔한 실수 → ## 같이 쓰면 좋은 기법`

**경제지표 글**: `## 무엇인가 → ## 왜 주가에 영향을 주나 → ## 언제 주목해야 하나 → ## 해석 기준 → ## 어디서 확인하나 → ## 함께 보면 좋은 지표`
- 지표 글에는 `indicator: {publisher, frequency, where}`와 `watch_when: [...]`(이럴 때 주목 3~6개) 필수.

## 문체·안전 원칙

- 문어체 평서형("~다"). 본문에 h1(#) 금지, ##부터 시작.
- **수익 보장·확실한 수익 암시 표현 절대 금지.** 고위험 기법(단타·레버리지)은 위험을 본문에서 명시.
- reliability는 검증 정도(1 참고용 ~ 5 장기 검증됨). 단기 트레이딩 기법은 3 이하.
- sources는 실존 서적·인물·기관만, URL 금지.

## 차트 데이터 (frontmatter `charts`)

- 교육용 가상 데이터를 직접 설계. candles 18~40개 `{o,h,l,c,v?}`, 반드시 `h>=max(o,c)`, `l<=min(o,c)`.
- 글에서 설명한 패턴이 캔들 흐름에 실제로 보여야 함. 오버레이: hlines/lines/zones/markers/sma (`src/components/ChartFigure.astro` 참조).
- 경제지표는 `kind: "line"` (candles의 c에 지표 값, o=h=l=c).
- 색 관행: 상승=빨강, 하락=파랑 (렌더러가 자동 처리).

## 검증

- 수정 후 반드시 `npm run build` — frontmatter가 zod 스키마로 검증되므로 빌드 통과가 1차 검증이다.

## 24시간 자동 수집 (스케줄 에이전트용 지침)

1. 기존 `src/content/techniques/` 파일 목록을 보고 **아직 없는** 기법/지표 1~2개를 선정 (또는 기존 글의 오류·구식 정보 업데이트).
2. 위 규칙대로 작성하되, 새 브랜치 `daily/<날짜>-<slug>`에 커밋하고 PR을 생성한다. **main에 직접 push 금지** — 사용자가 PR을 검토·머지해야 공개된다.
3. PR 본문에 요약과 검증 체크리스트(사실 확인, 차트 정합성, 빌드 통과)를 남긴다.
